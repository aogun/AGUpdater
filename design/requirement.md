# AGUpdater

这是一个自动更新系统，包含服务器，客户端的库和更新程序，客户端的库能够被其它程序调用提供自动更新功能。

## 服务器
- 服务器后端使用c++开发，构建系统使用 CMake，编译器为 Windows MinGW (MSYS2 mingw64)，前端页面使用 HTML/CSS/JavaScript/Vue3 开发，服务器需要提供RESTful API供客户端调用，接口需要使用HTTPS协议进行通信，服务器需要使用一个数据库来存储版本信息和下载统计信息，数据库使用SQLite数据库。
- 服务器针对管理和客户端提供两套接口，管理的接口需要登录，客户端使用的则不需要
- 服务器需要提供web页面供管理员管理，web页面需要登录，用户名和密码从配置文件中读取，功能如下
  - 能够查看已经上传的版本信息，删除指定版本，批量删除版本，修改版本信息等功能。
  - 进行版本文件的上传，上传时需要输入版本号，版本描述，上传对应压缩文件等信息，压缩包的格式为zip，上传时需要检查版本号是否已经存在，如果存在需要提示用户修改版本号后再上传，服务器需要检查压缩包的格式是否正确，文件大小是否超过限制（100MB）等，如果上传成功则需要将版本信息保存到服务器的数据库中，并且将文件保存到服务器的指定目录下。
  - 版本的信息展示需要包含版本号，版本描述，下载url，文件大小，文件sha256，上传时间，下载次数等信息。
  - 能够查看版本的下载统计信息，能够查看每个版本的下载来源和下载时间等信息。
- 客户端对接的接口的功能如下
  - 不需要登录，但是需要通过校验来检查是否合法客户端，校验的方法在后面会详细说明。
  - 提供查询指定版本之后是否有新版本的接口，查询最新版本信息的接口需要提供当前版本号，服务器会分析获取到指定版本之后的所有版本，将其作为列表返回，如果没有更新则返回没有更新的信息。
  - 提供下载指定版本的接口，下载接口需要提供版本号，服务器会返回对应版本的文件内容，如果版本号不存在则返回错误信息。
  - 提供查询所有版本信息的接口，接口需要提供分页参数，服务器会返回服务器上指定分页的版本的信息供客户端选择下载，如果没有版本信息则返回没有版本信息的提示。

## 客户端库
- 客户端分为3部分，一个是更新库，提供c++接口供其它程序调用，一个是有界面的版本管理工具，提供一个界面供用户查看服务器提供的所有版本和相关信息，并且能够选择下载和安装更新，第三部分是一个独立的更新程序，用于被库和版本管理工具调用，进行更新的下载和安装。
- 使用 C++ 开发，构建系统使用 CMake，编译器为 Windows MinGW (MSYS2 mingw64)
- CMakeLists.txt中通过编译选项来定义更新程序名，secret，服务器地址和端口，供第三方程序集成使用，更新库，版本管理工具和更新程序都会使用这些编译选项来获取服务器地址和端口等信息。
- 第三方程序集成时客户端时，采用源码集成的方式，用cmake的add_subdirectory来集成，集成后会生成一个静态库供其它程序链接使用，更新程序会被编译成一个独立的可执行文件，供库和版本管理工具调用。
- 库提供了查询更新接口供其它程序调用，接口是异步的，调用后会在后台进行版本检查和更新下载
  - 接口的输入参数是程序名称，当前版本号，回调函数，当前版本号用于服务器判断是否有更新，回调函数用于在检查更新完成后通知调用者检查结果。
  - 需要提供一个回调函数，回调函数会在检查更新完成后被调用，回调函数的输出参数至少有：
    - 调用服务器接口的错误码，成功为0，失败则是对应的错误码。
    - 如果检查更新成功，则会输出一个版本信息的结构体指针，可用于调用更新接口进行下载更新，包含版本号，版本描述，下载url，文件大小，文件sha256等信息，如果没有更新则输出空指针。
- 库提供下载更新接口供其它程序调用，接口是异步的，调用后会在后台进行更新下载
  - 接口的输入参数是版本信息结构体指针，回调函数，版本信息结构体指针用于指定要下载的版本，回调函数用于每秒更新下载进度，以及在下载完成后通知调用者下载结果，如果失败则通知调用者失败的错误码。
  - 下载文件放在系统临时目录下，下载完成后会返回下载文件的路径供调用者使用，如果下载失败则返回错误信息，并校验下载文件的sha256值是否正确，如果不正确则说明下载文件被篡改或者下载过程中出现了错误，需要删除下载文件并通过回调返回错误信息。
- 版本管理工具提供一个界面供用户查看服务器提供的所有版本和相关信息，并且能够选择下载和安装更新，界面需要包含以下功能：
  - 显示服务器上所有版本的信息，包含版本号，版本描述，下载url，文件大小，文件sha256，上传时间等信息。
  - 能够选择一个版本进行下载和安装，安装的过程需要显示下载进度和安装进度，如果下载或安装失败则显示错误信息。
  - 能够查看每个版本的详细信息，包括版本描述，下载url，文件大小，文件sha256等信息。
- 更新程序用来进行指定文件的解压和更新，文件为zip格式，zip中存在一个目录为程序名加版本号，将目录下的文件解压后覆盖更新程序所在目录下的文件除了更新程序自身

## 校验合法客户端的方案
- 客户端和服务器端都用同一个secret，服务器的secret可以通过配置文件读取，客户端的通过编译选项定义
- 校验的方法为客户端在每次调用接口时，包含身份校验的json字符串如下：
```json
{
  "device_id": "xxx",
  "timestamp": 1710748800,
  "nonce": "xxxx",
  "sign": "xxxx"
}
```
- device_id 是一个唯一标识客户端设备的字符串，是客户端从系统中获取的机器名称
- nonce 是一个随机数，每次请求都需要生成一个新的随机数，随机数的长度为16个字符，服务器会记录最近100个已经使用过的nonce，防止重放攻击
- sign 是一个HMAC值，使用secret作为密钥，device_id、timestamp和nonce作为消息计算出来的HMAC值，服务器会使用同样的方式计算出一个HMAC值并与客户端发送过来的sign进行比较，如果相同则说明请求合法，否则说明请求不合法。
- timestamp 是一个整数，表示请求发送的时间戳，单位为秒，服务器会检查这个时间戳是否在当前时间的10分钟之内，如果不在这个范围内则说明请求不合法。
- 服务器在接收到请求后会先进行校验，如果校验不通过则直接返回错误信息，如果校验通过则继续处理请求。
- 文件下载的接口也需要进行校验，校验的方法和上面一样，客户端在下载文件时需要在请求头中包含身份校验的json字符串，服务器会进行同样的校验，如果校验通过则返回文件内容，否则返回错误信息。

## 版本号
- 版本号采用语义化版本号，格式为主版本号.次版本号.修订号，例如1.0.0，2.1.3等。
- 主版本号表示重大更新，次版本号表示功能更新，修订号表示修复bug等小更新。
- 版本号需要唯一，不能重复，上传版本时需要检查版本号是否已经存在，如果存在需要提示用户修改版本号后再上传。
